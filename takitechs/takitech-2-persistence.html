JPA relations

  • OneToOne with its fetch type being eager
  • ManyToMany with its fetch type being lazy
  • OneToMany with its fetch type being lazy
  • ManyToOne with its fetch type being eager
    Eager means that the data will be fetched immediately, when Lazy means there will be a proxy on the data to be fetched later.

JPA

JPA is Java persistence API. It's a tool that helps us persist our data through the JAVA language.
JPA is an interface that uses rules and specifications to persist your data.
You can't use JPA by itself as it is an interface, but you can use its implementation, which is hibernate.

Note

You probably can't use non-relational databases with JPA lol

JDBC

JDBC means Java database connectivity.

EntityManager

EntityManager is an interface from JPA that helps interact with persistence's context, with the object lifecycle, the CRUD operations, etc...

Transactions

A transaction is a set of SQL operations following the ACID principle.

Lazy Loading

Lazy Loading creates a Proxy design pattern. Proxy means that the entity is created, but is an empty shell without any data in it.
The data is only fetched when explicitly asked for.

Pros and cons

PROS CONS
Performance and optimisation gain. If a transaction is not open when trying to fetch the proxy's data, an exception, LazyInitializationException, will be thrown.

ACID

  • Atomic: all operations are considered one and the same.
  • Coherence: you can either do everything or nothing: if the state of the operation becomes incoherent or false, you rollback everything.
  • Isolation: Two transactions cannot influence each other, they are independent.
  • Durability: As soon as the transaction is committed, it will be persisted.

JPA lifecycles

  • Transient: The entity has been created in java, but not yet pushed in the database
  • Persisted: The entity is present in the database.
  • Detached: The entity has been taken from the database, but isn't linked to the database through persistence's context. You use detached mode when you want to free transactions while doing a big operation on an entity. Then, you merge the entity when you're done doing the operation.
  • Removed: The entity is marked to be removed, but the change has not been yet persisted.

Hibernate

Hibernate is an object relational mapping (or ORM) and converts java objects into SQL entities. An alternative is EclipseLink.

Session

Session is Hibernate's entityManager. The difference is that you have more tools in Session than in entityManager, but has less portability, meaning you have to change all of the code if you don't want to use Session anymore, where you only have to change libraries if you use EntityManager.

N+1

When doing a lazy loading, if you have an entity tied to a many entity, you do a query when you want to fetch every single attached entity.
A solution is to use a join fetch or eager loading. You'd prefer a join fetch because eager loading is applied to every query.
