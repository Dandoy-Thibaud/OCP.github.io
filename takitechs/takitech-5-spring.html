Spring

Spring is a framework that helps create robust, easy-to-maintain and create java applications.

Dependency Injection

It is a design pattern that helps inject implementations from all around the code without needing to instantiate them.
You use constructors or @Autowired in tests to inject another bean.

Components

Your components are either a singleton where all components of the same type share the same instance, and prototype where they do not.
Components are an elemental brick of Spring. You can have multiple types of these components:

  • @Controller is your application's entrypoint. It doesn't hold any logic.
  • @Repository
  • @Service will be called by your entrypoint. It holds your logic.
  • @Component is the baseline of your components, every other component inherits this one.
    These annotations mostly don't do anything besides having a name, except @Controller that shows that it is an entrypoint for your dispatcher.

AOP

AOP is used to separate business and boilerplate code.
It helps avoid code duplication, and add behaviors on code without changing the base function

You can use AOP in different times: Spring is at runtime for example.

Proxies

Proxies are empty classes shells that don't hold any information to be added later.
You cannot use final or private classes for proxies.

How does Spring Work ?

You use a ComponentScan to scan for your components to then inject them where they are needed.

Spring makes your life easier

Spring takes care of the application's architecture:

  • SpringMVC will take care of the servlet for you with a servletdispatcher. A servlet is a web route.
    • Without it, you'd have to do the web configuration by hand.
  • Spring Boot helps by providing a pom.xml by default.
  • Boilerplate code is heavily reduced.
  • Spring boot comes with an embedded TomCat server, which helps not configure it.
    • The main difference with JEE is that servers come with everything, like a messaging library even if you don't need it for example. You can do everything, but it'll be heavy.

Spring vs Spring Boot

As explained, Spring Boot comes with a parent pom.xml that will pull the correct dependencies as needed.
It also comes with a heberger that comes with everything besides a JVM.

Inversion Of Control

IOC means that the framework will use the developper's code instead of the code using the framework's code.
For example, @Transactional is a IOC because it tells the framework to use a rollback at this point.

Java vs Kotlin

Java is retro-compatible when Kotlin is not.
Kotlin listens to its community as well.
It reduces boilerplate as it doesn't implement getters or setters.

Spring Modules

We have Spring core, Spring data, Spring security.

  • Spring Security deals with authentification, authorization, etc...
  • Spring Core is the Spring IOC container, where lies applicationContext and launches the componentScan. It manages your object's lifecycle, your configuration... It is not even imported, it is in by default.

@Transactional

See Takitech 2 - persistence
You can also have parameters, like readonly, like requiresNew...
It also implements a try catch as a rollback failsafe.

Exceptions

There are multiple types of exceptions:

  • Checked exceptions need to be flagged on a method, that the compiler will force you to handle.
    • For example: SQLException, IOException...
  • Unchecked exceptions are runtime exceptions that aren't anticipated.
    • For example: NullPointerException, IndexOutOfBoundsException...
