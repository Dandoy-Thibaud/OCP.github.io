Modelisation

Modelisation is done usually in UML, but this is falling out of fade quickly when the applications get bigger.

The main UML diagrams are:

  • comportement
    • Sequences, state-transition, use-case
  • Organisation
    • Classes, packages
  • Infrastructure
    • Deployment

Tools

  • BPMN 2.0: Process oriented
  • Archimate: to build UML efficiently, but you need to learn a lot to use it.

Context diagram

  • Gives an overview of the application
  • The main system is a colored box, in the middle.
  • Interactions with other systems and users are displayed.
  • It's more important to focus on the users, roles, personas and other systems that are dealt with on inferior levels.

What is C4 ?

C4 appeared when we realized we were trying to put stuff together that wasn't meant to.
You can see it as a map that you get closer and closer to, the amount of information that you need is not the same depending on the zoom level of the map.
C4 contains 4 views, ranked from most zoomed out to zoomed in:

  • Context
  • Container
  • Component
  • Classes

Classes

Regular UML class diagram.

Context

Describes all the macroscopic components, like the systems users can interact with.

Container

Describes how the application deploys, knowing the system requirements.

Container diagram

Container diagram is a zoom inside the colored box in the context diagram.
It focuses on the different systems inside the system, explaining all the different boxes that makes the bigger boxes.
Be sure to include:

  • Spread of responsibilities inside all the different elements of the system.
  • Technology choices inside the application.

Component

Describes the components inside a container.

Component diagram

A zoom on each building block.
It describes all the responsibilities inside the container diagrams.

Why C4 ?

It helps showing off the following aspects in a synthetic fashion quickly:

  • Actors
  • System components
  • Technologies
  • Interactions between actors and outside systems

C4's limits

  • There are no architecture elements in a C4 diagram, meaning there are no concepts of deployment.
    There's also no separation between software and the hardware that contains it.
  • It doesn't explain the logic of the application very well.

Tools

  • Templates available on Libre Office
  • Draw.io
  • Structurizr, helps automatically build the diagrams thanks to a direct link with you application's metadata.
